# 기여 가이드

브랜치 전략
- `master`: 릴리스용 안정 브랜치
- `feat/*`, `fix/*`: 주제별 작업 브랜치 → PR

커밋 메시지 컨벤션(권장)
- `feat:`, `fix:`, `docs:`, `refactor:`, `chore:` 접두사 사용

PR 체크리스트
- [ ] 동작 영향 범위 최소화(모듈 경계 유지)
- [ ] 상수/메시지/문서 동기화
- [ ] 콘솔 오류 0건
- [ ] CLAUDE.md/README 갱신

코드 스타일
- JSDoc 주석, 섹션 헤더, 명명 규칙(CLAUDE.md) 준수

테스트/검증
- 수동 시나리오 체크(탭 전환, 번역 중 취소, 지원 불가 URL)
