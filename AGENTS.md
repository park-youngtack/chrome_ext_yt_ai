# 에이전트 작업 지침

> ⚠️ **필수**: 작업 시작 전 반드시 `CLAUDE.md`를 먼저 읽고 프로젝트 전체 구조를 파악하세요.

## 기본 원칙

- 이 저장소의 모든 디렉터리와 파일에 대해 아래 지침을 적용합니다.
- 사용자와의 모든 커뮤니케이션은 한국어로 작성합니다.
- 코드 변경 시 주석은 반드시 한국어로 작성하고, 동작 원리를 개념적으로 설명해야 합니다.
- 재사용 가능한 구조를 우선하며, 하드코딩 대신 함수화/모듈화를 고려합니다.
- 테스트나 검증 명령을 실행했다면, 최종 응답에 동일한 명령을 이모지와 함께 명시합니다.
- PR 메시지를 작성할 때는 요약과 테스트 결과를 포함한 형식을 유지합니다.

## 프로젝트 참조 문서

**CLAUDE.md**는 프로젝트의 단일 진실 공급원(Single Source of Truth)입니다.
다음 정보는 `CLAUDE.md`에서 확인하세요:

- 프로젝트 아키텍처 및 파일 구조
- UI/UX 디자인 토큰 및 스타일 가이드
- 기능 명세 및 API 인터페이스
- 캐시 시스템 및 성능 최적화 전략
- 주요 작업 패턴 및 디버깅 팁

## 작업 전 체크리스트

- [ ] `CLAUDE.md` 읽기 (필수)
- [ ] `manifest.json` 버전 및 권한 확인
- [ ] `meta.js`의 `LAST_EDITED` 날짜 확인
- [ ] 작업 완료 후 `LAST_EDITED` 날짜 업데이트

## 금지 사항

- ❌ 페이지 오버레이 문구 추가
- ❌ 설정을 새 탭으로 열기
- ❌ 확장 이름 변경 (무조건 한글로 번역해드림)
- ❌ 패널 타이틀 변경 (AI 번역)
- ❌ 푸터 포맷 변경

## 필수 검증 항목

모든 작업 후 다음을 확인:

- [ ] 콘솔 에러 0건
- [ ] 우측 세로 탭바 정상 동작
- [ ] 딥링크 (#translate, #settings) 동작
- [ ] 세션 복원 동작
- [ ] 캐시 TTL 및 재검증 동작
- [ ] 접근성 (ARIA, 키보드 내비게이션)

## 커밋 메시지 규칙

- `feat:` 새 기능
- `fix:` 버그 수정
- `refactor:` 리팩토링
- `style:` 스타일 변경
- `docs:` 문서 변경
